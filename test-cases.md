# テストケース一覧

## 目的

筋トレ履歴表示アプリが、仕様通りに動作しているかを確認する。  
各テストケースは、機能IDと対応させて管理する。

---

## テストケース

| TC-ID | 機能ID | テスト内容             | 操作手順                                                             | 期待結果                                                     | 実際の結果 | OK/NG |
| ----- | ------ | ---------------------- | -------------------------------------------------------------------- | ------------------------------------------------------------ | ---------- | ----- |
| TC-01 | F-01   | 正常系：筋トレ履歴表示 | 1. `npm run dev` を実行する<br>2. `http://localhost:3000` を開く     | 筋トレ履歴がカード形式で表示される                           |            |       |
| TC-02 | F-02   | 正常系：API確認        | 1. `http://localhost:3000/api/training-records` を開く               | 筋トレ履歴のJSONデータが表示される                           |            |       |
| TC-03 | F-03   | loading表示            | 1. `http://localhost:3000` を開く<br>2. 読み込み直後の表示を確認する | データ取得中に「読み込み中...」が表示される                  |            |       |
| TC-04 | F-03   | empty表示              | 1. APIが空配列を返す状態にする<br>2. `http://localhost:3000` を開く  | 「まだ記録がありません。」と表示される                       |            |       |
| TC-05 | F-03   | error表示              | 1. APIのURLを一時的に間違える<br>2. `http://localhost:3000` を開く   | 「エラー」「データの取得に失敗しました」が表示される         |            |       |
| TC-06 | F-04   | コンポーネント分割確認 | 1. `src/components` を確認する                                       | `TrainingList.tsx` と `TrainingCard.tsx` が存在する          |            |       |
| TC-07 | F-05   | UI表示確認             | 1. `http://localhost:3000` を開く                                    | 筋トレ履歴がカード形式で表示され、部位ごとに色分けされている |            |       |

---

## 手動テスト時の注意

- 期待結果は Yes / No で判定できる内容にする
- 実際の結果には、確認した内容を短く記録する
- OK/NG欄には、確認後に OK または NG を記入する
