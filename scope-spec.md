# 機能一覧（Day5版 Scope SPEC）

## 目的

筋トレ履歴表示機能について、正常系・非正常系・Out of Scope を整理し、テスト可能な仕様にする。

---

## 機能一覧

| 機能ID | 機能名                       | 正常系                                                                    | 非正常系                                                                       | Out of Scope                      |
| ------ | ---------------------------- | ------------------------------------------------------------------------- | ------------------------------------------------------------------------------ | --------------------------------- |
| F-01   | 筋トレ履歴表示               | トップページを開くと、APIから取得した筋トレ履歴がカード形式で表示される   | APIから空配列が返った場合、「まだ記録がありません。」を表示する                | 履歴の検索・並び替え・フィルター  |
| F-02   | API連携                      | `/api/training-records` にアクセスすると、筋トレ履歴のJSONが返る          | API取得に失敗した場合、画面にエラーメッセージを表示する                        | 認証付きAPI・ユーザー別データ取得 |
| F-03   | loading / error / empty 表示 | データ取得中は「読み込み中...」を表示し、取得後に履歴を表示する           | 取得失敗時は「データの取得に失敗しました」を表示し、0件時は空状態を表示する    | 高度なローディングアニメーション  |
| F-04   | コンポーネント分割           | `page.tsx`、`TrainingList.tsx`、`TrainingCard.tsx` に役割を分けて表示する | コンポーネントに渡すデータが不正な場合は画面崩れの原因になるため、型定義で防ぐ | Atomic Designの完全な階層化       |
| F-05   | UI改善                       | 筋トレ履歴をカード形式で表示し、部位ごとに色の違いを出す                  | データが欠けている場合、表示内容が不自然になる可能性がある                     | レスポンシブ対応・高度なデザイン  |

---

## In Scope（今回実装する機能）

| 機能ID | 機能名             | 完了定義                                                  |
| ------ | ------------------ | --------------------------------------------------------- |
| F-01   | 筋トレ履歴表示     | トップページで筋トレ履歴がカード形式で表示される          |
| F-02   | API連携            | `/api/training-records` でJSONデータが返る                |
| F-03   | 状態表示           | loading / error / empty の3状態が実装されている           |
| F-04   | コンポーネント分割 | `TrainingList.tsx` と `TrainingCard.tsx` に分割されている |
| F-05   | UI改善             | カードUIと部位ごとの色分けが表示される                    |

---

## Out of Scope（今回は実装しない機能）

| 機能名                 | 理由                                                  |
| ---------------------- | ----------------------------------------------------- |
| ログイン認証           | 今回はUIとAPI接続、テスト可能性の確認を優先するため   |
| ユーザー別データ管理   | 認証機能を実装しないため                              |
| 筋トレ記録の新規登録   | まずは履歴表示の安定動作を優先するため                |
| 筋トレ記録の編集・削除 | CRUD全体ではなく、表示機能に絞るため                  |
| AIおすすめ提案         | MVPの範囲外であり、履歴表示完成後の拡張機能とするため |
| グラフ分析             | 実装時間が大きくなるため、今回は対象外とする          |
| 通知機能               | 継続支援として有効だが、今回の実装範囲外とする        |

---

## 今回のMVP

今回のMVPは、筋トレ履歴をAPIから取得し、Reactの画面にカード形式で表示できる状態とする。

完成条件：

- `/api/training-records` がJSONを返す
- トップページで筋トレ履歴が表示される
- loading / error / empty が表示される
- UIがコンポーネント分割されている
- 仕様・実装・テストが対応している
