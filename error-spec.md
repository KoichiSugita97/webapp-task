# 非正常系・バリデーション SPEC

## 目的

- アプリが正常に動かない場合の表示や処理を定義する
- ユーザーが原因を理解できるようにする
- API取得失敗やデータ0件時でも画面が崩れないようにする

---

## Empty状態

### ケース

- 筋トレ履歴が1件もない
- 初回利用でまだ記録が存在しない

### 表示内容

- 「まだ記録がありません。」

### 完了定義

- APIから空配列が返った場合に、空状態メッセージが表示される

---

## Loading状態

### ケース

- APIからデータを取得中

### 表示内容

- 「読み込み中...」

### 完了定義

- データ取得中に loading 表示が出る

---

## Error状態

### ケース

- API取得に失敗した
- ネットワークエラーが発生した
- サーバー側でエラーが発生した

### 表示内容

- 「エラー」
- 「データの取得に失敗しました」

### 完了定義

- APIレスポンスが正常でない場合にエラーメッセージが表示される

---

## バリデーション

### 今回対象

現在は履歴表示が中心のため、入力フォームは実装しない。

### 今後追加する場合

- weight は 0 より大きい数値
- reps は 0 より大きい整数
- sets は 0 より大きい整数
- exerciseName は空文字不可
- trainingPart は空文字不可

---

## 今回実装しない非正常系

| 項目                       | 理由                     |
| -------------------------- | ------------------------ |
| ログイン切れ               | 認証機能を実装しないため |
| 他ユーザーのデータアクセス | userId固定のため         |
| 記録削除失敗               | 削除機能を実装しないため |
